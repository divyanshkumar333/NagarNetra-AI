# Contributing to NagarNetra AI

## Branching Strategy
- `main`: Stable production branch.
- `develop`: Integration branch for features.
- `feature/*`: New features and components.

## Code Standards
- Strict TypeScript must be used. No `any`.
- Keep components under 300 lines. Break them down if they get larger.
- Use reusable hooks and utility functions from `lib/` and `hooks/`.
- No duplicated code.
- Write concise but meaningful comments.
